# 排七 Web 游戏轻奢会所 UI 重构设计文档

## 1. 背景

当前版本已经具备可玩的规则引擎和基础对局流程，但视觉和信息架构存在两个根本问题：

- 扑克牌不像真实扑克牌，更像抽象游戏卡片。
- 对局页的信息层级过于面板化，主舞台不够纯粹，导致“牌桌感”很弱。

这次改版不改变任何核心规则，也不改 AI 规则策略。目标是把产品从“可玩的原型”提升为“像高端会所牌桌的成品”。

## 2. 产品结论

本轮重构的产品结论如下：

- 整体风格切换为 `轻奢会所`。
- 背景采用 `深色绒面牌桌`，扑克牌是视觉主角。
- 对局页收敛为 `顶部极简状态 + 中部四花色主舞台 + 底部一排玩家手牌`。
- 四个花色区内采用 `7 居中，小牌向上，大牌向下` 的竖向接龙结构。
- 顶部仅保留必要状态与 `AI 手牌数`，不再保留常驻大事件流和多余信息卡。
- 玩家交互集中在底部一排真实手牌，采用 `直线轻叠牌`，兼顾真实感和可点击性。
- 非法牌仍然可见，但被弱化；合法牌通过抬升、提亮、边缘强调形成主焦点。
- 最近动作通过桌内 `短暂提示` 展示，不再使用常驻事件列表。
- 首页、规则弹窗、结算页统一重做，保持同一视觉语言。
- 对局主页面走 `Pixi-first`，非对局页面仍由 React 负责结构与内容。

## 3. Definition of Done

当以下条件同时成立时，本轮 UI 重构视为完成：

- 截图状态下，用户一眼就能识别这是一款真实扑克牌驱动的高端牌桌产品，而不是泛游戏卡片界面。
- 对局页结构被压缩为三层：顶部极简状态、中部四花色牌位、底部一排玩家手牌。
- 扑克牌正反面、背牌、合法态、非法态、选中态属于同一套真实扑克视觉系统。
- 去掉大事件流后，用户仍能从动画和短暂提示理解“轮到谁、出了什么、借到了什么”。
- 桌面端和手机端都能顺畅完成完整牌局，且不会因为真实牌叠放而变得难以操作。

## 4. 范围边界

### 4.1 本轮要做

- 首页重做
- 对局页重做
- 规则弹窗重做
- 结算页重做
- 扑克牌渲染系统重做
- 对局信息架构重做
- 电影感动效重做

### 4.2 本轮不做

- 不改排七规则
- 不改 AI 难度逻辑
- 不新增账号、排行、皮肤、商店
- 不接入联机
- 不引入有版权风险的外部品牌扑克牌素材

## 5. 视觉方向

### 5.1 核心关键词

- 轻奢会所
- 深色绒面
- 象牙白纸牌
- 克制金属感
- 稳定、安静、贵
- 电影感而非赛博感

### 5.2 视觉原则

- 扑克牌必须成为第一视觉主角。
- 背景和容器为牌服务，不能比牌更抢眼。
- 高级感来自比例、材质、留白、层级，而不是大面积发光描边。
- 动画是注意力引导，不是装饰噪音。

### 5.3 色彩与材质

- 主背景：深蓝黑或深墨绿绒面桌面。
- 容器：低对比、弱边界、细腻阴影。
- 牌面：暖白、象牙白、细边框。
- 牌背：深色底配精致纹样，可带极轻金属感。
- 强调色只用于合法动作、关键反馈和按钮焦点，不能成为整页主色。

## 6. 扑克牌视觉系统

### 6.1 牌面规范

- 使用标准扑克牌长宽比。
- 角标包含点数与花色。
- 红桃与方块使用红色体系，黑桃与梅花使用黑色或深色体系。
- 数字牌与人头牌都必须符合真实扑克牌认知。
- 正面、背面、缩略、手牌态、桌面态必须共用统一设计。

### 6.2 状态规范

- 合法牌：轻微抬起、亮度更高、边缘更清晰。
- 非法牌：保持真实牌样，但亮度更低、位置更安静。
- Hover / Focus：只做克制的阴影和位移反馈。
- 禁止把非法牌做成另一种“UI 卡片”。

### 6.3 背牌规范

- AI 背牌出现在顶部极简区域，数量少，主要用于建立对桌感。
- 背牌样式必须比当前 “SEVENS” 文字卡更真实、更统一。

## 7. 页面结构

### 7.1 首页

首页的职责不是堆功能，而是建立牌桌氛围与品牌第一印象。

- 中央主视觉应展示真实扑克牌和高端牌桌质感。
- 标题、难度选择、开始按钮位于同一视觉中心。
- 规则说明和音效开关是二级操作。
- 首页与对局页切换后不能像进入另一个产品。

### 7.2 对局页

对局页采用三层结构：

1. 顶部极简状态带
2. 中部四花色主舞台
3. 底部玩家手牌带

常驻大事件流、行动面板、统计卡片全部移除。

### 7.3 规则弹窗

- 像桌边规则卡，而不是普通后台式 modal。
- 内容短、排版克制、重点明确。
- 与首页和对局页保持同一材质和字体系统。

### 7.4 结算页

- 像一局牌结束后的收桌时刻。
- 保留胜负结果、简短结算信息和再来一局入口。
- 不做复杂统计大屏。

## 8. 对局页详细结构

### 8.1 顶部极简状态带

只保留：

- 当前轮到谁
- 双方手牌数
- 返回首页
- 顶部 AI 少量背牌装饰

不再保留多块大信息卡，也不保留常驻事件流。

### 8.2 中部四花色主舞台

四块花色区域是对局页绝对主角。

每个花色区域具备：

- 花色标题
- 居中的 `7` 锚点
- 上方小牌接龙区
- 下方大牌接龙区
- 未开线时的克制占位提示

每个区域看起来应像桌面上的牌位，而不是业务面板。

### 8.3 底部玩家手牌带

- 玩家手牌是一排横向轻叠牌。
- 视觉上保留真实拿牌感，但点击区域必须清晰。
- 合法牌默认抬起或更亮。
- 非法牌更暗，但仍属于同一排真实手牌。

### 8.4 短暂动作反馈

最近动作通过牌桌内短暂提示完成：

- “轮到你”
- “AI 打出红桃 8”
- “借到方块 7”
- “立即出牌”

提示出现后自动淡出，不再长期占用布局。

## 9. 动效设计

### 9.1 目标

动效目标是建立 `电影感`，不是炫技。

### 9.2 关键动作

- 发牌：从牌堆方向发出，带重量感和停顿。
- 出牌：从底部手牌提起后进入对应花色区。
- 借牌：从顶部背牌区抽出一张，短暂停顿后决定入手牌或直接落桌。
- 结算：以收桌、聚焦胜负或轻量镜头强调结束，而不是粗暴弹框。

### 9.3 节奏约束

- 动效需要统一节奏。
- 高频动作不能拖慢牌局。
- 牌移动画优先服务“牌去了哪里”的理解。

## 10. 技术架构

### 10.1 总体路线

本轮采用 `Pixi-first`：

- 对局页由 Pixi 主导渲染。
- React 负责页面壳、路由状态、规则弹窗、首页和结算页结构。
- 规则引擎和 AI 不改，继续复用现有 TypeScript 核心。

### 10.2 为什么不是纯 DOM

因为这次明确需要更强电影感，且用户接受 `Pixi-first` 路线。  
因此对局主舞台、真实手牌、短暂提示和关键动画应放入同一渲染系统，避免 DOM 与 Canvas 分裂。

### 10.3 对局场景分层

建议将对局页 Pixi 场景拆为：

- `BackdropLayer`
- `TopStatusLayer`
- `OpponentLayer`
- `SuitBoardLayer`
- `TransientFeedLayer`
- `PlayerHandLayer`
- `CinematicFxLayer`

各层职责单一，避免再次堆成一层大组件。

### 10.4 牌组件系统

需要建立统一的扑克牌绘制与布局组件：

- `PokerCardSprite`
- `CardBackSprite`
- `HandRailLayout`
- `SuitStackLayout`
- `TransientToast`

真实牌感应来自统一组件系统，而不是各处手画小卡片。

### 10.5 交互策略

- 视觉重叠与点击热区解耦。
- 点击非法牌不抛错、不跳大提示，只做轻反馈。
- 真正可行动作仍由规则层决定，UI 只是表达更高级。

## 11. 响应式与可用性

### 11.1 桌面端

- 强调完整牌桌气质。
- 四花色区域和底部手牌拥有充分留白。
- 顶部状态带更薄更克制。

### 11.2 手机端

- 保持三层结构不变。
- 底部手牌重叠程度略降，确保点击。
- 四花色区仍保留上下接龙逻辑，不改规则表达。

### 11.3 可访问性

- 合法动作不能只靠颜色区分。
- 合法牌需同时具备亮度、位置或边缘变化。
- 动画时长要控制，避免眩晕。

## 12. 风险与对策

### 12.1 风险：做成“高配赛博 UI”

对策：
- 严禁用大面积霓虹描边当高级感。
- 以材质、牌面、比例和留白为主。

### 12.2 风险：真实手牌影响点击

对策：
- 点击热区与视觉重叠分离。
- 桌面和手机分别调叠牌密度。

### 12.3 风险：Pixi-first 导致首页/弹窗掉档

对策：
- 非对局页面继续使用 React + CSS，但视觉系统与对局页一致。

## 13. 验收映射

### 13.1 结构验收

- 对局页一眼分成三层。
- 四花色区成为绝对主角。

### 13.2 牌面验收

- 任何人都应认出这是真实扑克语法，而非抽象卡片。

### 13.3 信息验收

- 去掉侧栏事件流后仍能理解对局推进。

### 13.4 风格验收

- 首页、对局页、规则弹窗、结算页属于同一产品。

### 13.5 操作验收

- 合法牌清楚、非法牌不误导、手机端可点。

## 14. 后续产出

基于本设计文档，下一步输出实施计划，按以下原则执行：

- 先改信息架构，再改牌组件，再做动效。
- 先让结构正确，再叠加高级感。
- 每一步都保持桌面和手机端可玩。
