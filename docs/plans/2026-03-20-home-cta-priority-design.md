# 首页 CTA 首屏优先设计

**Goal:** 调整首页的信息层级和响应式策略，让 `开始游戏` 在手机、平板、桌面 Web 首屏都可见，并消除 iPad 上内容被裁切且无法下滑的问题。

**Context:**
- 当前首页把 `开始游戏` 放在模式说明之后，主操作在信息层级上靠后。
- `src/index.css` 只在 `899px` 以下放开纵向滚动；更宽的平板会继续使用 `overflow: hidden` 与固定 `100dvh` 高度。
- `src/App.css` 中首页的双栏布局从 `1080px` 开始启用，但平板横屏与部分窄桌面仍会承受较高的垂直内容密度。
- 当前需求明确要求保留首页全部内容，不新增悬浮 CTA，不修改“姓名为空时禁用开始按钮”的规则。

**Design:**
- 首页主信息区重排为：标题/简介 -> 玩家姓名 -> 难度选择 -> 主操作区 -> 模式说明。
- `开始游戏` 保持为最显著的主操作，并在所有设备下优先出现在首屏中段；`规则说明` 和 `音效开关` 继续作为次操作跟随出现。
- 首页滚动策略由“仅手机可滚动”改为“首页在任意视口都允许纵向滚动”，避免平板和窄桌面因为固定高度导致内容被截断。
- 调整首页断点，使平板宽度优先服务主操作可见性，而不是过早进入高密度桌面排版；装饰性牌桌继续保留，但不能把左侧 CTA 挤出首屏。

**Non-Goals:**
- 不修改游戏页、结果页和规则弹层的交互布局。
- 不修改开始游戏前的校验逻辑、音效逻辑或难度逻辑。
- 不删除首页既有文案和装饰区域，只做顺序和响应式调整。

**Validation:**
- 增加首页结构测试，锁定 `开始游戏` 位于难度选择之后、模式说明之前。
- 增加 E2E 视口测试，验证 `开始游戏` 在手机、iPad 竖屏、iPad 横屏、桌面视口首屏内可见。
- 运行相关 Vitest 与 Playwright 测试，确认首页仍能正常输入姓名、切换难度、开始对局。
